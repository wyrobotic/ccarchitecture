Design Workflow
Default Path — Single Pass
1
PRD — describe what it does, not how it looks
No colors, shadows, layouts, or visual specs
↓
2
/prepare → /architect
Architect MUST hand off to designer unless pattern is an exact replica
↓
3
@pact-frontend-designer — researches functional intent, brand context, design intelligence → produces complete spec
↓
4
@pact-frontend-coder — implements from spec
OR
Opt-In — Sandbox Exploration
1
"I don't know what I want yet"
Genuinely exploring directions
↓
2
/sandbox-design — pure aesthetics, no tech stack constraints, ui-ux-pro-max skill
↓
3
Pick a direction → approved mockup becomes reference
↓
4
Feed into Default Path step 2
Key Principle
The frontend designer is a skilled professional, not an implementor of your visual specs. Describe the functional problem — let the designer research the right aesthetic solution using design intelligence, brand context, and the existing design system.
